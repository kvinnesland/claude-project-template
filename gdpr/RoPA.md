# Records of Processing Activities (RoPA)

> Article 30 GDPR — legally required record of all processing activities.
> Update this file whenever a CR introduces, modifies, or removes a processing activity.
> Last updated: [date]

---

## How to Use This File

Each processing activity gets one entry below.
A "processing activity" is a distinct purpose for which personal data is used.
One product typically has 5–20 processing activities.

---

## Entry Template

```
### [Activity name]

| Field | Value |
|---|---|
| Purpose | [Why is this data processed?] |
| Legal basis | [Art. 6 basis — and Art. 9 basis if sensitive data] |
| Data subjects | [Who: users, employees, visitors, etc.] |
| Personal data categories | [What fields: name, email, IP, behavioral, etc.] |
| Sensitive data (Art. 9) | [Yes / No — if yes, specify category and Art. 9(2) exception] |
| Source | [Direct from user / inferred / third party] |
| Recipients | [Internal teams / processors / other controllers] |
| Third-country transfers | [Yes / No — if yes: country, safeguard (SCC, adequacy, BCR)] |
| Retention period | [How long is data kept, and why] |
| Deletion mechanism | [How and when is it deleted — automated or manual process] |
| Security measures | [Encryption at rest, access control, pseudonymization, etc.] |
| DPIA required | [Yes / No — if yes: link to DPIA document] |
| CR reference | [CR-XXX that introduced this activity] |
| Last reviewed | [Date] |
```
